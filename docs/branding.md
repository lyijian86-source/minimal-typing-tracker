# Branding Direction

## Recommended Repository Name

`type-ledger`

## Why

- Searchable: includes the product's core action, `type`
- Memorable: `ledger` gives it a distinctive product identity
- Trustworthy: sounds measured, private, and calm
- Extensible: still fits if the product grows into richer analytics

## Secondary Options

- `type-pulse`
- `key-ledger`
- `typing-ledger`

## Naming Guidance

Use `TypeLedger` as the public-facing GitHub/release brand while keeping internal migration from `Type Record` gradual.

Recommended order:

1. Rename the GitHub repository to `type-ledger`
2. Keep code/package identifiers stable for now
3. Migrate UI/app strings only when release packaging is ready
